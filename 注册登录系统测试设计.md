# 注册登录系统测试设计

## 系统概述

一个用于软件测试练习的完整用户注册登录系统，包含前端界面和后端API。

## 系统架构

```
前端 (HTML/CSS/JavaScript)
    ↓
API接口 (RESTful API)
    ↓
后端服务 (Java/Python)
    ↓
数据库 (MySQL/PostgreSQL)
```

## 功能模块

### 1. 用户注册模块

#### 1.1 注册页面
- 用户名输入框（4-20位字母数字）
- 邮箱输入框（有效邮箱格式）
- 密码输入框（8-20位，包含大小写字母和数字）
- 确认密码输入框
- 验证码输入框（4位数字）
- 用户协议复选框
- 注册按钮

#### 1.2 注册流程
1. 用户填写注册信息
2. 前端验证输入格式
3. 发送验证码到邮箱
4. 输入验证码验证
5. 提交注册请求
6. 注册成功跳转到登录页

#### 1.3 注册规则
- 用户名唯一性检查
- 邮箱唯一性检查
- 密码强度验证
- 验证码有效期5分钟
- 用户协议必须同意

### 2. 用户登录模块

#### 2.1 登录页面
- 用户名/邮箱输入框
- 密码输入框
- 记住我复选框
- 忘记密码链接
- 登录按钮
- 注册链接

#### 2.2 登录方式
1. 用户名+密码登录
2. 邮箱+密码登录
3. 第三方登录（可选）

#### 2.3 登录状态
- 登录成功跳转到首页
- 登录失败显示错误信息
- 记住我功能（7天免登录）
- 登录次数限制（5次失败锁定15分钟）

### 3. 用户管理模块

#### 3.1 个人中心
- 个人信息展示
- 头像上传
- 基本信息修改
- 密码修改
- 安全设置

#### 3.2 安全功能
- 修改密码（需要原密码）
- 绑定手机号
- 邮箱验证
- 登录记录查看
- 异常登录提醒

## API接口设计

### 1. 注册相关API

#### POST /api/register/check-username
检查用户名是否可用
```json
请求: {"username": "testuser"}
响应: {"available": true, "message": "用户名可用"}
```

#### POST /api/register/send-code
发送验证码到邮箱
```json
请求: {"email": "user@example.com"}
响应: {"success": true, "message": "验证码已发送"}
```

#### POST /api/register
用户注册
```json
请求: {
  "username": "testuser",
  "email": "user@example.com",
  "password": "Test123456",
  "confirmPassword": "Test123456",
  "verificationCode": "1234",
  "agreeTerms": true
}
响应: {"success": true, "message": "注册成功", "userId": 123}
```

### 2. 登录相关API

#### POST /api/login
用户登录
```json
请求: {
  "username": "testuser",
  "password": "Test123456",
  "rememberMe": false
}
响应: {
  "success": true,
  "token": "jwt_token_here",
  "userInfo": {
    "userId": 123,
    "username": "testuser",
    "email": "user@example.com"
  }
}
```

#### POST /api/logout
用户登出
```json
请求: {"token": "jwt_token_here"}
响应: {"success": true, "message": "登出成功"}
```

### 3. 用户管理API

#### GET /api/user/profile
获取用户信息
```json
响应: {
  "userId": 123,
  "username": "testuser",
  "email": "user@example.com",
  "avatar": "avatar_url",
  "createdAt": "2026-04-07"
}
```

#### PUT /api/user/profile
更新用户信息
```json
请求: {
  "username": "newusername",
  "email": "newemail@example.com"
}
响应: {"success": true, "message": "更新成功"}
```

#### PUT /api/user/password
修改密码
```json
请求: {
  "oldPassword": "Old123456",
  "newPassword": "New123456"
}
响应: {"success": true, "message": "密码修改成功"}
```

## 自动化测试脚本示例

### 1. Selenium Web自动化测试

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.common.keys import Keys
import time

class TestRegistrationLogin:
    
    def setup(self):
        self.driver = webdriver.Chrome()
        self.driver.implicitly_wait(10)
        self.base_url = "http://localhost:8080"
        
    def test_valid_registration(self):
        """测试有效注册"""
        driver = self.driver
        driver.get(f"{self.base_url}/register")
        
        # 填写注册信息
        driver.find_element(By.ID, "username").send_keys("testuser001")
        driver.find_element(By.ID, "email").send_keys("test001@example.com")
        driver.find_element(By.ID, "password").send_keys("Test123456")
        driver.find_element(By.ID, "confirmPassword").send_keys("Test123456")
        driver.find_element(By.ID, "verificationCode").send_keys("1234")
        driver.find_element(By.ID, "agreeTerms").click()
        
        # 点击注册按钮
        driver.find_element(By.ID, "registerBtn").click()
        
        # 验证注册成功
        time.sleep(2)
        assert "登录" in driver.title
        print("注册测试通过")
        
    def test_invalid_password(self):
        """测试无效密码"""
        driver = self.driver
        driver.get(f"{self.base_url}/register")
        
        # 填写弱密码
        driver.find_element(By.ID, "password").send_keys("weak")
        driver.find_element(By.ID, "registerBtn").click()
        
        # 验证错误提示
        error_msg = driver.find_element(By.CLASS_NAME, "error-message").text
        assert "密码长度不足" in error_msg
        print("密码验证测试通过")
        
    def teardown(self):
        self.driver.quit()

if __name__ == "__main__":
    test = TestRegistrationLogin()
    test.setup()
    test.test_valid_registration()
    test.test_invalid_password()
    test.teardown()
```

### 2. API测试脚本（使用requests）

```python
import requests
import json

class APITest:
    
    def __init__(self):
        self.base_url = "http://localhost:8080/api"
        self.session = requests.Session()
        
    def test_registration_api(self):
        """测试注册API"""
        url = f"{self.base_url}/register"
        
        # 测试数据
        data = {
            "username": "apitestuser",
            "email": "apitest@example.com",
            "password": "ApiTest123",
            "confirmPassword": "ApiTest123",
            "verificationCode": "1234",
            "agreeTerms": True
        }
        
        response = self.session.post(url, json=data)
        result = response.json()
        
        assert response.status_code == 200
        assert result["success"] == True
        print("注册API测试通过")
        
    def test_login_api(self):
        """测试登录API"""
        url = f"{self.base_url}/login"
        
        data = {
            "username": "apitestuser",
            "password": "ApiTest123",
            "rememberMe": False
        }
        
        response = self.session.post(url, json=data)
        result = response.json()
        
        assert response.status_code == 200
        assert result["success"] == True
        assert "token" in result
        print("登录API测试通过")
        
        # 保存token供后续测试使用
        self.token = result["token"]
        
    def test_profile_api(self):
        """测试获取用户信息API"""
        url = f"{self.base_url}/user/profile"
        headers = {"Authorization": f"Bearer {self.token}"}
        
        response = self.session.get(url, headers=headers)
        result = response.json()
        
        assert response.status_code == 200
        assert "username" in result
        assert "email" in result
        print("用户信息API测试通过")

if __name__ == "__main__":
    api_test = APITest()
    api_test.test_registration_api()
    api_test.test_login_api()
    api_test.test_profile_api()
```
